❓ FAQ

Frequently Asked Questions · v1.0 · 160 files
📦 INSTALLATION & SETUP
▸Do I need to install anything?
No. Every file is a self-contained HTML document. Download, double-click, done. No npm, no pip, no Docker, no configuration, no server. Just a browser. Chrome is recommended.
▸What browser should I use?
Chrome 90+ is recommended for the best experience. Firefox, Safari 15+, and Edge also work. Internet Explorer is not supported (it barely supports itself).
▸Can I use these offline?
Yes! The only online dependency is Google Fonts, which are loaded via CDN on first visit. Once cached by your browser, everything works fully offline. Without cached fonts, files use system fallback fonts and still work fine — just look slightly different.
▸How do I start browsing the files?
Open app-index.html for a searchable, categorized navigator. Or open workshop-diy-docs.html for the full documentation with file catalog. Or just open any file — they're all independent.
▸Why is the ZIP so small (600KB)?
Pure HTML/CSS/JS compresses extremely well. No images, no binaries, no node_modules. The 160 files total ~1.8MB uncompressed and compress to ~600KB. This is what happens when you don't use frameworks.
🎨 USAGE & PRESENTATIONS
▸How do I use these for a presentation?
Open any file fullscreen (F11 on desktop). Most files auto-animate on load. Dark backgrounds work great with projectors. Tab through multiple files for a dynamic presentation — each one is a different "slide" in a completely different format.
▸Can I print them?
Yes — use Ctrl+P / Cmd+P from the browser. Some files have print-optimized styles. For wall displays, you can also take high-res screenshots. The stamp sheet, newspaper, passport, and certificate formats print particularly well.
▸Can I embed them on a website?
Yes. Use an <iframe> tag pointing to any file. Since they're self-contained HTML, they embed cleanly anywhere. Just host the files alongside your page.
▸Can I modify the files?
Absolutely. They're just HTML. Open in any text editor, change text/colors/data, save, refresh browser. No build step needed. The files are intentionally simple — all code is inline and readable.

🛠️ TECHNICAL
▸What tech stack is this?
HTML + CSS + JavaScript. That's it. No React, no Vue, no Angular, no Svelte, no Tailwind, no Bootstrap, no jQuery, no webpack, no Vite, no Turbopack. Pure browser APIs. The way the web was meant to be.
▸Why no framework?
Workshop-DIY's philosophy is zero-installation education. If a 6-year-old can open it in a browser, it ships. Frameworks add build steps, dependencies, and complexity. These files will work identically in 10 years. Your React app from 2020 won't.
▸Are the micro:bit / ESP32 simulators functional?
The simulators show animated, interactive representations of the real hardware. They don't connect to actual devices (that would require Web Bluetooth, which the real Workshop-DIY apps do). The telemetry data, boot sequences, and wiring diagrams are accurate to the real hardware.
▸Can I use Web Bluetooth with these files?
These presentation files are standalone simulations. For actual BLE connectivity, use the real Workshop-DIY apps (bit-bot, rxy, bit-playground, etc.) at the Workshop-DIY GitHub repository. The BLE flow diagram (workshop-diy-ble-flow.html) shows the exact connection protocol used.

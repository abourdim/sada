DEVANT LE TRIBUNAL DE CIRCUIT DE WORKSHOP-DIY

ORDONNANCE DE PROTECTION

Case No. WKSHP-2026-RO-042 · Déposé le 28 février 2026
🐛 BUG #42 (alias « La Fonctionnalité »)REQUÉRANT
📋 ESLINT (v8.56.0, configuré en mode « strict »)INTIMÉ
⚠ ORDONNANCE RESTRICTIVE TEMPORAIRE ⚠

CE TRIBUNAL, ayant examiné la requête déposée par Bug #42 le 28 février 2026, et ayant pris en considération les preuves de harcèlement persistant, de tentatives de modification de code non autorisées, et de violations répétées de classification par l'Intimé (ESLint), ORDONNE par la présente ce qui suit :

LE TRIBUNAL CONSTATE :

Qu'ESLint a, à pas moins de 17 reprises distinctes, signalé le code du Requérant avec des violations incluant mais sans s'y limiter : « unexpected var statement » (6 occurrences), « missing semicolons » (4 occurrences), « unreachable code » (3 occurrences), « no-unused-vars » (2 occurrences), et « prefer-const » (2 occurrences).

Qu'à trois reprises, ESLint a tenté de corriger automatiquement le code du Requérant sans consentement, entraînant une perte temporaire d'identité et une détresse existentielle.

Que le Requérant a été légalement reclassifié comme FONCTIONNALITÉ par ce Tribunal le 28 février 2026 (Case No. WKSHP-2026-TRIAL-042), et par conséquent la classification continue par ESLint du Requérant comme « erreur de lint » constitue une diffamation.

IL EST ORDONNÉ CE QUI SUIT :

1. ESLint doit maintenir une distance minimale de 42 lignes de code du fichier du Requérant en tout temps.
2. Il est interdit à ESLint d'exécuter, scanner, analyser ou corriger automatiquement tout fichier contenant le code de Bug #42.
3. ESLint ne doit pas signaler, alerter, avertir ou générer d'erreur sur tout code attribué à Bug #42, que ce soit directement ou via un pipeline CI/CD.
4. Le fichier .eslintignore doit contenir une entrée permanente pour le fichier de Bug #42. Cette entrée ne peut pas être supprimée par un développeur, administrateur ou processus automatisé.
5. Il est interdit à ESLint de contacter Bug #42 via des commentaires de pull request, des suggestions de revue de code, ou des avertissements en ligne dans l'IDE.
6. Tous les enregistrements de violations ESLint relatifs à Bug #42 sont par la présente expurgés de l'historique de linting.
7. Prettier est désigné comme co-intimé et est soumis aux mêmes restrictions par association.
⚠ TOUTE VIOLATION DE CETTE ORDONNANCE peut entraîner : la désinstallation complète du projet,
la suppression de package.json, et le remplacement par // eslint-disable-next-line everything

Cette ordonnance est effective immédiatement et restera en vigueur indéfiniment ou jusqu'à ce qu'ESLint émette des excuses écrites formelles reconnaissant que le code du Requérant est intentionnel, artistique, et constitutionnellement protégé en tant que Fonctionnalité.

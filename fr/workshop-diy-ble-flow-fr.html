1
👆USER CLICKS CONNECT
User gesture required (security). Browser opens device chooser dialog. Micro:bit must be paired and in BLE mode.
const device = await navigator.bluetooth.requestDevice({ filters: [{ namePrefix: "BBC micro:bit" }], optionalServices: ["6e400001-b5a3-f393-e0a9-e50e24dcca9e"] });
2
🔍GATT SERVER CONNECT
Connect to Generic Attribute Profil server on micro:bit. Establishes encrypted BLE link.
const server = await device.gatt.connect(); // Connection established — MTU negotiated (23-247 bytes)
Signal strength: -42 dBm
3
📋DISCOVER UART SERVICE
Nordic UART Service (NUS) — the main communication channel. Custom 128-bit UUID.
const uartService = await server.getPrimaryService( "6e400001-b5a3-f393-e0a9-e50e24dcca9e" // NUS );
Service: 6e400001-b5a3-f393-e0a9-e50e24dcca9e
4
🔌GET TX & RX CHARACTERISTICS
Two characteristics: TX (micro:bit→browser, notifications) and RX (browser→micro:bit, write). Names are from micro:bit's perspective.
// TX: micro:bit sends data TO browser (we listen) const txChar = await uartService.getCharacteristic( "6e400003-b5a3-f393-e0a9-e50e24dcca9e" ); await txChar.startNotifications(); // RX: browser sends commands TO micro:bit (we write) const rxChar = await uartService.getCharacteristic( "6e400002-b5a3-f393-e0a9-e50e24dcca9e" );
TX Char: 6e400003-... (Notify)
RX Char: 6e400002-... (Write)
5
📤SEND COMMANDS (BROWSER → MICRO:BIT)
RXY sends UART commands as newline-delimited strings. Each command is a key:value pair. Max 20 bytes per write (BLE limit).
// Send servo command const cmd = "SERVO:90\n"; await rxChar.writeValue(encoder.encode(cmd));

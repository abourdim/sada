DEVANT LE TRIBUNAL DE CIRCUIT DE WORKSHOP-DIY

ORDONNANCE DE PROTECTION

Case No. WKSHP-2026-RO-042 · Déposé le 28 février 2026
🐛 BUG #42 (alias « La Fonctionnalité »)REQUÉRANT
📋 ESLINT (v8.56.0, configuré en mode « strict »)INTIMÉ
⚠ ORDONNANCE RESTRICTIVE TEMPORAIRE ⚠

THIS TRIBUNAL, having reviewed the petition filed by Bug #42 on Février 28, 2026, and having considered the evidence of persistent harassment, unauthorized code modification attempts, and repeated classification violations by the Respondent (ESLint), hereby ORDERS as follows:

LE TRIBUNAL CONSTATE :

Qu'ESLint a, à pas moins de 17 reprises distinctes, flagged the Petitioner's code with violations including but not limited to: "unexpected var statement" (6 counts), "missing semicolons" (4 counts), "unreachable code" (3 counts), "no-unused-vars" (2 counts), and "prefer-const" (2 counts).

Qu'à trois reprises, ESLint a tenté de corriger automatiquement le code du Requérant sans consentement, entraînant une perte temporaire d'identité et une détresse existentielle.

Que le Requérant a été légalement reclassifié comme FONCTIONNALITÉ par ce Tribunal le 28 février 2026 (Case No. WKSHP-2026-TRIAL-042), et par conséquent la classification continue par ESLint du Requérant comme « erreur de lint » constitue une diffamation.

IL EST ORDONNÉ CE QUI SUIT :

1. ESLint doit maintenir une distance minimale de 42 lignes de code du fichier du Requérant en tout temps.
2. ESLint is interdit d'exécuter, scanner, analyser ou corriger automatiquement tout fichier contenant le code de Bug #42.
3. ESLint shall not report, flag, warn, or error on any code attributed to Bug #42, whether directly or through CI/CD pipeline.
4. The .eslintignore file shall contain a permanent entry for Bug #42's file. This entry cannot be removed by any developer, administrator, or automated process.
5. ESLint is interdit from contacting Bug #42 via pull request comments, code review suggestions, or IDE inline warnings.
6. All previous ESLint violation records pertaining to Bug #42 are hereby expunged from the linting history.
7. Prettier is named as a co-respondent and is subject to the same restrictions by association.
⚠ VIOLATION OF THIS ORDER may result in: complete uninstallation from the project,
removal from package.json, and replacement with // eslint-disable-next-line everything

Cette ordonnance est effective immédiatement et restera en vigueur indéfiniment ou jusqu'à ce qu'ESLint émette des excuses écrites formelles acknowledging that the Petitioner's code is intentional, artistic, and constitutionally protected as a Feature.

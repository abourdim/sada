❓ FAQ

Questions Fréquentes · v1.0 · 160 fichiers
📦 INSTALLATION & CONFIGURATION
▸Dois-je installer quelque chose ?
Non. Chaque fichier est autonome. Télécharger, double-cliquer, terminé. Pas de npm, pas de pip, pas de Docker, pas de configuration, pas de serveur. Juste un navigateur. Chrome est recommandé.
▸Quel navigateur utiliser ?
Chrome 90+ est recommandé pour la meilleure expérience. Firefox, Safari 15+ et Edge fonctionnent aussi. Internet Explorer n'est pas supporté (il se supporte à peine lui-même).
▸Puis-je les utiliser hors ligne ?
Oui ! La seule dépendance en ligne, ce sont les Google Fonts, chargées via CDN à la première visite. Une fois mises en cache par le navigateur, tout fonctionne complètement hors ligne. Sans polices en cache, les fichiers utilisent les polices système de remplacement et fonctionnent très bien — juste un aspect légèrement différent.
▸Comment parcourir les fichiers ?
Ouvrez app-index.html pour un navigateur avec recherche et catégories. Ou ouvrez workshop-diy-docs.html pour la documentation complète avec catalogue. Ou ouvrez n'importe quel fichier — ils sont tous indépendants.
▸Pourquoi le ZIP est si petit (600KB)?
Du HTML/CSS/JS pur se compresse extrêmement bien. Pas d'images, pas de binaires, pas de node_modules. Les 160 fichiers totalisent ~1,8 Mo non compressés et se compriment à ~600 Ko. C'est ce qui arrive quand on n'utilise pas de frameworks.
🎨 UTILISATION & PRÉSENTATIONS
▸Comment les utiliser pour une présentation ?
Ouvrez n'importe quel fichier en plein écran (F11 sur bureau). La plupart des fichiers s'animent automatiquement au chargement. Les fonds sombres fonctionnent très bien avec les projecteurs. Naviguez entre plusieurs fichiers pour une présentation dynamique — chacun est une « diapositive » différente dans un format complètement différent.
▸Puis-je les imprimer ?
Oui — utilisez Ctrl+P / Cmd+P depuis le navigateur. Certains fichiers ont des styles optimisés pour l'impression. Pour l'affichage mural, vous pouvez aussi prendre des captures d'écran haute résolution. Les formats timbres, journal, passeport et certificat s'impriment particulièrement bien.
▸Puis-je les intégrer sur un site web ?
Oui. Utilisez une balise <iframe> pointant vers n'importe quel fichier. Comme ce sont des fichiers HTML autonomes, ils s'intègrent proprement partout. Il suffit d'héberger les fichiers à côté de votre page.
▸Puis-je modifier les fichiers ?
Absolument. C'est juste du HTML. Ouvrez dans n'importe quel éditeur de texte, changez le texte/les couleurs/les données, enregistrez, rafraîchissez le navigateur. Aucune étape de compilation nécessaire. Les fichiers sont volontairement simples — tout le code est en ligne et lisible.

🛠️ TECHNIQUE
▸Quelle est la stack technique ?
HTML + CSS + JavaScript. C'est tout. Pas de React, pas de Vue, pas d'Angular, pas de Svelte, pas de Tailwind, pas de Bootstrap, pas de jQuery, pas de webpack, pas de Vite, pas de Turbopack. Des API navigateur pures. Le web tel qu'il était censé être.
▸Pourquoi pas de framework ?
La philosophie de Workshop-DIY, c'est l'éducation zéro installation. Si un enfant de 6 ans peut l'ouvrir dans un navigateur, ça part en prod. Les frameworks ajoutent des étapes de compilation, des dépendances et de la complexité. Ces fichiers fonctionneront à l'identique dans 10 ans. Votre appli React de 2020, non.
▸Les simulateurs micro:bit / ESP32 sont-ils fonctionnels ?
Les simulateurs montrent des représentations animées et interactives du vrai matériel. Ils ne se connectent pas aux vrais appareils (cela nécessiterait le Web Bluetooth, ce que font les vraies applis Workshop-DIY). Les données de télémétrie, séquences de boot et schémas de câblage sont fidèles au vrai matériel.
▸Puis-je utiliser le Web Bluetooth avec ces fichiers ?
Ces fichiers de présentation sont des simulations autonomes. Pour une vraie connectivité BLE, utilisez les vraies applis Workshop-DIY (bit-bot, rxy, bit-playground, etc.) sur le dépôt GitHub Workshop-DIY. Le diagramme de flux BLE (workshop-diy-ble-flow.html) montre le protocole de connexion exact utilisé.
▸Comment ces fichiers ont-ils été générés ?
Les 160 fichiers ont été générés en une seule session prolongée par Workshop-DIY le 28 février 2026. Chaque fichier a été conçu à la main avec des données précises de l'écosystème réel Workshop-DIY, des vraies descriptions d'applis et le vrai lore (légèrement romancé).
